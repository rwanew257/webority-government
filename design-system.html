Introduction #

Three stylesheets define everything below. None of them are bundled or compiled — they are hand-authored and loaded in this order on every page.

FilePurpose
css/style.cssTokens (verbatim from the parent firm’s core.min.css) and the base component library
css/gov-identity.cssGovernment-identity bar, skip-nav link, focus-visible outline
css/design-polish.cssLayout polish, hero v2, stats v2, all the *-v2 / *-v3 variants and gov-specific cards
Principle: tokens are copied verbatim from www.webority.com/css/bundles/core.min.css. The two sites share a design language — do not invent new tokens here. Add component-level CSS in design-polish.css rather than mutating style.css.

Buttons #

Buttons are how the user commits to an action. The system has one base class (.btn), seven types, fluid sizes that scale across breakpoints, and five interaction states. Defaults are correct for almost every situation — reach for a modifier only when the visual hierarchy demands it.

The single most-used button on the site is the navigation CTA <a class="btn btn-primary">Contact Us &rarr;</a>. It appears in the header on every page. Render it exactly as shown:
Contact Us →

Anatomy

A button is a label, optionally preceded or followed by an icon, inside a pill-shaped container. The container is always horizontally centred with consistent vertical and horizontal padding.

Request a Consultation → container · .btn label (Josefin Sans 600) icon / arrow (optional)

The default CTA. Purple gradient fill, white label. Use for the single most important action on a section — usually exactly one per view (header CTA aside).

Secondary — .btn .btn-secondary

Dark-navy fill, white label. Reach for this only on white sections where a second high-emphasis action sits next to a primary — rarely needed.

Transparent fill, purple border + label. Lower-emphasis alternative; fills purple on hover.

Pick a size, then pick a type. The size modifier (.btn-lg / .btn-md / .btn-sm) controls dimensions only. The type modifier (.btn-primary, .btn-outline, etc.) controls colour and emphasis. They are orthogonal and combine freely.

Do’s & Don’ts

Do Use .btn .btn-primary for every primary action, including the header CTA. The right-arrow &rarr; is part of the label, not a separate icon.
Don’t Stack .btn with .btn-pill — they are separate components with conflicting padding, font-size and letter-spacing rules.
Do Render CTAs as <a> when they navigate, and <button type="submit/button"> when they trigger script or form submission.
Don’t Override font-family, font-weight or border-radius with inline styles. If a one-off variant is needed, add a class to design-polish.css.
Do Use .btn-glow on the homepage hero only. The animated highlight loses meaning when used in every section.
Don’t Place two .btn-primary buttons in the same section — pair a primary with an outline or ghost variant to preserve hierarchy.
Do Use .btn-outline-light exclusively on dark or photographic backgrounds. The default .btn-outline assumes a light surface.
Don’t Wrap a button label. Keep CTA copy to 1–4 words. If the action needs more context, use the lead paragraph above it.
